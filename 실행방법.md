

# 서버 실행 방법

## 1. 서버 폴더 이동

```bash
cd server
```

---

## 2. 가상환경 생성 (최초 1회)

```bash
python3 -m venv venv
```

생성 확인:

```bash
ls -la
```

`venv` 폴더가 존재해야 한다.

---

## 3. 가상환경 활성화

```bash
source venv/bin/activate
```

정상 활성화 시 터미널에 `(venv)` 또는 사용하는 쉘의 가상환경 표시가 나타난다.

---

## 4. 의존성 설치

프로젝트 루트에 `pyproject.toml`이 존재하는 경우:

```bash
pip install -e .
```

설치 확인:

```bash
pip list
```

---

## 5. 서버 실행

```bash
./scripts/dev.sh
```

---

# 자주 발생하는 오류

## source: no such file or directory: .venv/bin/activate

원인:

- `.venv` 폴더가 존재하지 않음
- 실제 가상환경 이름이 `venv`일 가능성이 높음

확인:

```bash
ls -la
```

활성화:

```bash
source venv/bin/activate
```

---

## No module named uvicorn

원인:

- uvicorn이 설치되지 않음

확인:

```bash
pip list | grep uvicorn
```

설치:

```bash
pip install uvicorn
```

또는

```bash
pip install -e .
```

---

## No module named fastapi

원인:

- 프로젝트 의존성이 설치되지 않음

설치:

```bash
pip install -e .
```

확인:

```bash
pip list | grep fastapi
```

---

## ERROR: [Errno 48] Address already in use

원인:

- 동일한 포트를 사용하는 서버가 이미 실행 중임

확인:

```bash
lsof -i :8000
```

프로세스 종료:

```bash
kill -9 <PID>
```

또는:

```bash
lsof -ti :8000 | xargs kill -9
```

실행 중인 uvicorn 확인:

```bash
ps aux | grep uvicorn
```

---
